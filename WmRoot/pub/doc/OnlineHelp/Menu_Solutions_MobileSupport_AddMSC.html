Add Mobile Sync Component Screen
Screen Heading: Mobile Support > Mobile Sync Components > Add Mobile Sync Component
Use the Add Mobile Sync Component screen to add a mobile sync component.
To open the Add Mobile Sync Component screen, click Mobile Support under the Solutions menu on the Navigation panel, and then click Mobile Sync Components. Click Add Mobile Sync Component on the Mobile Sync Components screen.
Click Return to Mobile Sync Components to return to the previous screen.
Click the Save Changes button to save changes you made.
Mobile Sync Component
Field...
Specify...
Alias
A unique name for the mobile sync component.
Download Service
The fully qualified name of the flow service created to download data from the backend database to the mobile device.
Upload Service
The fully qualified name of the flow service created to upload data from the mobile device to the backend database.
Business Doc Type
The fully qualified name of the IS document type to use to define the structure of the data to be synchronized.
Mobile Support uses this document type to construct the mDataRow variable in the wm.mobile.datasync.specs:downloadSpec and wm.mobile.datasync.specs:uploadSpec specifications implemented in the wm.mobile.datasync:synchronize service.
Row Identifiers
One or more field names from the business document type that you want to use to uniquely identify the row. Separate field names with commas.
Conflict Resolution Rule
The rule that Mobile Support should use when evaluating whether conflicts exist for new synchronization requests. Select one of the following:
Client wins: Accept data the mobile device uploads, even if changes were made to that data row in the backend database since the device last synchronized with the server.
Server wins: Ignore the data the mobile device uploads if changes were made to the data in the backend database since the device last synchronized with the server.
